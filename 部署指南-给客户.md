# TeableSync 安装部署指南

## 简介

TeableSync 是一款将 SQL Server / MySQL / PostgreSQL 数据同步到 Teable 的工具，带 Web GUI 管理界面。

---

## 系统要求

| 项目 | 要求 |
|------|------|
| Node.js | >= 18 |
| 操作系统 | Windows / macOS / Linux |
| SQL 数据库 | SQL Server、MySQL、PostgreSQL 任选其一 |
| Teable | 自部署版或官方云版（app.teable.ai） |

---

## 快速部署（5分钟完成）

### 第一步：下载项目

**方式一：Git 克隆**
```bash
git clone https://github.com/jclee8810-max/teable-sync.git
cd teable-sync
```

**方式二：直接下载 ZIP**
访问 https://github.com/jclee8810-max/teable-sync/releases 下载最新版本

---

### 第二步：安装依赖

```bash
# 一键安装全部依赖
npm install

# 如果网络慢，可以用淘宝镜像
npm install --registry=https://registry.npmmirror.com
```

---

### 第三步：启动服务

```bash
npm run dev
```

启动成功后显示：
```
TeableSync Server → http://localhost:3100
TeableSync UI     → http://localhost:5173
```

打开浏览器访问 **http://localhost:5173** 即可看到管理界面。

---

### 第四步：配置连接

#### 添加 SQL Server 连接

1. 点击「连接管理」→「添加连接」
2. 选择类型：**SQL Server**
3. 填写配置：

| 字段 | 示例值 |
|------|--------|
| 连接名称 | 测试SQL Server |
| Host | localhost 或 192.168.x.x |
| Port | 1433 |
| 数据库 | SyncPilotTest |
| 用户名 | sa |
| 密码 | QClaw2026!Strong |

4. 点击「测试连接」→ 显示「连接成功」→ 保存

#### 添加 Teable 连接

1. 点击「连接管理」→「添加连接」
2. 选择类型：**Teable**
3. 填写配置：

| 字段 | 示例值 |
|------|--------|
| 连接名称 | 测试Teable |
| URL | http://localhost:3000 |
| Token | 你的Teable Token |

> Teable Token 获取方式：Teable 右上角 → 账号设置 → API Token

4. 点击「测试连接」→ 显示「连接成功」→ 保存

---

### 第五步：创建同步任务

1. 点击「同步任务」→「新建任务」
2. 选择源连接（SQL Server）→ 选择源表（如 Employees）
3. 选择目标连接（Teable）→ 选择目标表（如 Employees Sync）
4. 字段映射自动检测，可按需调整
5. 同步模式选择「手动」或「定时」
6. 冲突策略建议选择「Upsert」（按主键更新，不重复插入）
7. 保存

---

### 第六步：执行首次同步

1. 在任务列表找到刚创建的任务
2. 点击「▶ 执行」按钮
3. 观察底部日志面板，看到「同步完成」即成功

**成功标志：**
- 日志显示「全量拉取 X 条」
- 日志显示「新增 X 条，更新 Y 条」
- Teable 目标表中已能看到同步过来的数据

## 生产环境部署

### 构建前端

```bash
npm run build
```

### 使用 PM2 启动后端服务

```bash
# 安装 PM2
npm install -g pm2

# 启动服务（后台运行）
cd server
pm2 start src/index.js --name teable-sync

# 设置开机自启
pm2 save
pm2 startup
```

### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        proxy_pass http://127.0.0.1:5173;
    }

    # 后端 API
    location /api {
        proxy_pass http://127.0.0.1:3100;
    }

    # WebSocket 日志
    location /ws {
        proxy_pass http://127.0.0.1:3100;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## 常见问题

### Q: 连接测试失败？

**检查清单：**
1. SQL Server 是否已启动，端口 1433 是否可达
2. 用户名/密码是否正确
3. 数据库是否已创建
4. 防火墙是否放行了端口

### Q: 同步拉取 0 条记录？

原因：增量同步时，时间戳列的值都比上次同步时间早。
**解决方法：** 删除 `server/data/sync-state/` 目录下的状态文件（或者改文件名备份），再重新同步即为全量同步。

### Q: Teable Token 在哪里获取？

1. 登录 Teable
2. 点击右上角头像 → 「账号设置」
3. 找到「API Token」→ 复制

### Q: 支持 Teable 官方云版吗？

支持。在添加 Teable 连接时，URL 填 `https://app.teable.ai`，Token 填你的云版 API Token 即可。

### Q: 支持 Docker 部署吗？

支持！项目已包含 `docker-compose.yml`，可一键启动：

```bash
# 下载 Docker 部署包（客户提供）
# 启动
docker-compose up -d

# 打开管理界面
open http://localhost:5173
```

包含 TeableSync + SQL Server（测试用）。如已有外部数据库，改 docker-compose.yml 里地址即可。
